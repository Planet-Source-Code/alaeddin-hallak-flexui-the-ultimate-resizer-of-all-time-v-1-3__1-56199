1. Overview


     Programmers have to deal with load of repetitive tasks in their work everyday. They may come up with automotive solutions to some of them, but others remain pain-in-the-neck. One of these tasks is having to handle the resize event of a form with some or many controls. Many programmers revert to preventing user from being able to resize the from altogether by making form border unresizable. While this is preferred setting for dialog boxes, it is certainly not for main application windows and windows that contain variable amount of data that can be better viewed when the form is enlarged. For this and many other reasons, FlexUI helps to ease your pain and automate this process to a high level.

     FlexUI is a lightweight ActiveX control that can be used in any environment that supports OLE automation. FlexUI provides means for the programmer to specify, at design time, how each control should respond to form's resize event, called control behaviour. Control behaviour can be any possible combination of anchoring and resizing operations.

     In addition, FlexUI supports form right-to-left effect by, optionally, inverting the control the style of control behaviour when the form's RightToLeft property is set to True. Finally, the control allows you to set minimum and maximum width and height of the form and prevents resizing the form beyond those limits.

2. Installation


     If you have downloaded this component from a website as an installation package, simply run the setup file. Otherwise, you'll have to register the control before you can use it. To do that, follow these steps:

  1. Copy FlexUI.ocx to a folder of your choice (preferably your system folder)
  2. Click Start > Run
  3. Type regsvr32 "path" where path is the where you placed FlexUI.ocx (including the file name)

3. Using FlexUI

    3.1 Adding FlexUI:

To start using FlexUI, you must add it to your application. In Visual Basic 6, the way to do this is by going to Project > Components and selection FlexUI from the list of available controls. Now you can add an instance of FlexUI to any form from the toolbox.

    3.2 Setting Controls' Behaviour at design-time:

Right-click on the FlexUI instance in your form and choose Properties. This will open up a dialog similar to the one shown below:

In this dialog, you can set each control's behaviour by first choosing the control from the list on the left and then choosing a behaviour from the list on the right. Optionally, you can check Reverse anchor behaviour for this control in Right-To-Left mode checkbox. Doing so will cause FlexUI to calculate the reverse behaviour and use it when form's RightToLeft property is set to True. (Note: you must call the Resize procedure of the control after you have set the form's RightToLeft property for the changes to take effect)

    3.3 Setting and Getting Control's Behaviour at Runtime

In addition to design-time, FlexUI also allows you to set or get a control's behaviour at runtime. Controls behaviour is stored as variable of an an enumeration called AnchorResizeStyle. To retrieve the value of a controls behaviour, call GetControlStyle passing the control for which you want to retrieve the style as a parameter.

Set a control's behaviour using SetControlStyle method, passing the control along with the new style value as parameters. Note that to specify RightToLeft effect along with a chosen style use + operator. For example, BottomAnchor_RightAnchor + RTLEffect.  This method is useful when you dynamically create controls at runtime.

    3.4 Specifying Minimum and Maximum Width and Height for t he Container Form

FlexUI allows you to optionally specify a minimum or maximum width or height for the form it's placed on. To do this, select the FlexUI instance in the properties window. Change the appropriate properties to the value you want (in screen pixels). If you want the control to ignore the value, type in -1.

    3.5 Update Control's Size or Position Manually

If you happen to change a control's size or position at runtime, you must call UpdateControlPositionSize passing the changed control as parameter. Not doing so may result in unpredictable anchoring and resizing of the control by FlexUI.

    3.6 Backup/Restore Controls Behaviour Configuration

If your form has many controls, you may find it safe to backup controls' behaviour in case FlexUI gets deleted off the form or for any other reason you might think of. Later, you can restore the configuration by selecting the same file you backup to.
To backup controls' configuration:

1.       Right-click on the FlexUI instance in your form and choose Properties

2.       Switch to Back/Restore tab

3.       Click Backup

4.       Specify a file to backup to

To restore a backed up configuration:

1.       Right-click on the FlexUI instance in your form and choose Properties

2.       Switch to Back/Restore tab

3.       Click Restore

4.       Specify a file to restore from.

5.       You must click OK for changes to take effect.

4. Known Issues

There is a number of know issues in FlexUI that you should be aware of:

* If you notice that the anchor/resize behaviour of certain controls is lost when switching to runtime, try cutting and repasting the controls and then specifying an anchor/resize behaviour for them. This problem is due to a deficiency in VB.

* Currently, FlexUI does not support controls that don't support Container property, such as Data control. We hope to add support for those controls in future versions.

* Call UpdateControlPositionSize method after you manually resize or reposition a control. Not doing so may result in unpredictable anchoring or resizing behaviour by FlexUI.
